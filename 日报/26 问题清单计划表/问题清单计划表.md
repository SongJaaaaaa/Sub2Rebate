# 2026-06-26 问题清单 & 实施计划

## 一、返利余额提现方式扩展

### 1.1 现状

当前提现只有一条路：「返利余额 → 申请提现 → 管理员审核 → 人工支付宝打款」。用户绑定支付宝账号后发起提现，管理员在后台点 approve → paid 完成流程。

### 1.2 需求目标

提现应支持两种出口：

| 方式 | 去向 | 到账速度 | 审核 |
|------|------|----------|------|
| A. 转入 Sub2API 额度 | Sub2API 用户余额 | 秒级 | 可免审/自动 |
| B. 支付宝自动打款 | 用户支付宝账户 | 秒级~分钟级 | 看金额和风控策略 |

---

### 1.3 方式 A：返利余额 → Sub2API 额度

**原理**：直接调用已有的 `Sub2ApiAdminClient::updateUserBalance()`，将返利余额扣减，同时给 Sub2API 账户加额度。

**改动清单**：

| 层 | 文件/位置 | 改动 |
|----|-----------|------|
| 数据库 | withdraw_records migration | 新增 `type` 字段：`alipay`（默认）/ `api_quota` |
| 后端 Service | `WithdrawService` | 新增 `applyToApiQuota()` 方法；扣减返利余额 → 调 Sub2ApiAdminClient 加额度 → 记录流水 |
| 后端 Controller | `WithdrawController` | 新 route `POST /v1/withdraw/to-api-quota`，或在现有 apply 接口加 `type` 参数 |
| 后端配置 | config_items | 新增 `withdraw.to_api_quota_enabled` 开关、`withdraw.to_api_quota_rate`（换算比例，默认 1:1） |
| 前端 | `WithdrawView.vue` | 提现方式 Tab 切换：「提现到支付宝」/「转入 API 额度」；后者不需要绑定支付宝账号 |
| 审计 | audit_logs | 记录 `withdraw_to_api_quota` 事件 |

**核心流程**：

```
用户选择「转入 API 额度」
  → 输入金额
  → 后端校验余额 ≥ 金额
  → DB 事务：扣减 rebate_balance.available_amount，创建 withdraw_record(type=api_quota, status=paid)
  → 调用 Sub2ApiAdminClient::updateUserBalance(user.sub2api_id, amount, 'increment', 'Sub2Rebate 余额转入')
  → 成功 → 返回新余额
  → 失败 → 回滚事务，提示用户重试
```

**关键设计决策**：
- 转入 API 额度是「即时完成」，不走审核流，因为钱没出系统，只是从返利余额挪到 API 额度。
- 换算比例默认 1:1，后台可调（应对运营活动打折场景）。
- 需要做幂等处理（idempotency key 使用 `withdraw_record.id`）。

### 1.4 方式 B：支付宝自动打款

**参考 Epay 的实现**：Epay 代付模块 `includes/lib/Transfer.php` 通过支付宝 `alipay.fund.trans.uni.transfer` 接口实现免密转账到支付宝账户。核心参数：

```
product_code: TRANS_ACCOUNT_NO_PWD
biz_scene: DIRECT_TRANSFER
payee_info.identity: 收款支付宝账号
payee_info.identity_type: ALIPAY_LOGON_ID / ALIPAY_USER_ID
payee_info.name: 收款人真实姓名（用于校验）
trans_amount: 金额
```

**前置条件**（签约要求）：
- 支付宝开放平台创建应用，签约「现金红包」或「单笔转账到支付宝账户」能力
- 应用公钥证书模式（推荐）或普通密钥模式
- 需做身份报备（transfer_scene_name + report_infos），否则大额会被拦截

**改动清单**：

| 层 | 文件/位置 | 改动 |
|----|-----------|------|
| 后端新 Service | `Modules/Payment/Services/AlipayTransferService.php` | 封装 `alipay.fund.trans.uni.transfer` 和 `alipay.fund.trans.common.query` |
| 后端配置 | config_items | 新增 `payment.alipay_transfer.enabled`、`app_id`、`private_key`（或证书路径）、`alipay_public_key`、`daily_limit_amount`、`single_max_amount` |
| 后端 WithdrawService | `markPaid()` 改造 | 管理员点「打款」时自动调用 AlipayTransferService；或开启 auto_pay 模式后审批通过即自动打款 |
| 后端新 Command | `ProcessAutoPayoutCommand` | 定时捡取 status=approved 的提现单，执行自动打款；失败则标记 retry |
| 数据库 | withdraw_records | 新增 `payout_trade_no`（支付宝转账流水号）、`payout_error`（失败原因）、`payout_time` |
| 后端 Admin | `AdminWithdrawController` | 在 approve 后根据配置自动触发打款或保留手动 |
| 前端 Admin | `AdminWithdrawalsView.vue` | 新增「自动打款」开关状态展示、打款失败重试按钮、转账流水号列 |

**核心流程（自动打款模式）**：

```
管理员审批通过
  → 判断 auto_pay 是否开启
    → 是：调用 AlipayTransferService::transfer(account, realName, amount)
      → 成功 → 记录 payout_trade_no，状态改为 paid
      → 失败（收款方原因）→ 标记 payout_error，状态保持 approved，等管理员处理
      → 失败（系统原因）→ 入重试队列
    → 否：保持当前手动流程
```

**安全与风控**：
- 单笔限额：后台可配（默认 500 元）
- 日累计限额：后台可配（默认 5000 元）
- 同一账号同日打款次数限制
- 打款前再次验证收款人姓名与绑定账号一致
- 所有打款操作写 audit_log

**分期建议**：
- 第一阶段：先做「管理员点按钮触发自动打款」（半自动），验证接口通了
- 第二阶段：再做 auto_pay 模式（审批即打款）+ 定时重试 Command

## 五、支付宝自动打款签约与技术要点

### 5.1 签约路径

1. 登录 [支付宝开放平台](https://open.alipay.com)
2. 创建网页应用 → 添加「单笔转账到支付宝账户」功能
3. 选择公钥证书模式（推荐），下载三个证书文件：
   - 应用公钥证书 `appCertPublicKey.crt`
   - 支付宝公钥证书 `alipayCertPublicKey_RSA2.crt`
   - 支付宝根证书 `alipayRootCert.crt`
4. 完成身份报备：营业执照信息 + 场景说明

### 5.2 技术参考（来自 Epay）

Epay 的 `alipay_plugin.php` 的 `transfer()` 方法核心逻辑：

```php
// 证书模式
$bizContent = [
    'out_biz_no' => $outBizNo,       // 唯一流水号
    'trans_amount' => $money,         // 金额
    'product_code' => 'TRANS_ACCOUNT_NO_PWD',
    'biz_scene' => 'DIRECT_TRANSFER',
    'order_title' => '返利提现',
    'payee_info' => [
        'identity' => $alipayAccount,
        'identity_type' => 'ALIPAY_LOGON_ID', // 或 ALIPAY_USER_ID
        'name' => $realName,           // 收款人真实姓名（验证）
    ],
];
// 调用 alipay.fund.trans.uni.transfer
```

### 5.3 Sub2Rebate 中的封装方案

新建 `AlipayTransferService`，不依赖 Epay 系统，独立封装：
- 使用 Laravel 的 `alipaysdk`（如 `yansongda/pay`）或自行封装 RSA2 签名
- 证书存放在 `storage/certs/alipay/`
- 配置从 config_items 动态读取
- 结果异步通知走 `POST /v1/withdraw/alipay/transfer-notify`（可选）

## 六、风险与注意事项

1. **支付宝签约周期**：单笔转账能力审核需 1-3 个工作日，建议提前发起
2. **证书安全**：私钥不可入库，生产环境通过环境变量或文件挂载
3. **幂等**：每笔打款必须用唯一 out_biz_no，推荐用 withdraw_record.id 前缀
4. **对账**：定期用 `alipay.fund.trans.common.query` 比对本地状态和支付宝状态
5. **资金安全**：自动打款必须有限额兜底，防止代码 bug 导致资金大量流出
6. **Sub2API 额度转入的不可逆性**：一旦转入 API 额度，不支持逆向转回返利余额（需在 UI 明确提示）
