# Risk 模块：风控与防刷

## 1. 模块目标

负责降低刷量、异常邀请、异常提现和财务风险。

## 2. 模块边界

本模块负责：

- 黑名单。
- 异常邀请检测。
- 最低提现限制。
- 冻结期。
- 风控标记。

本模块不负责：

- 返利计算。
- 支付回调。

## 3. MVP 风控

第一版至少实现：

- 黑名单用户不可提现。
- 最低提现金额。
- 同一事件不重复发放返利。
- 管理员可冻结用户提现。

## 4. 后续风控

- IP / 设备异常。
- 自邀请检测。
- 高频小额充值。
- 批量小号。
- 异常提现。

## 5. 已完成

- 模块文档初始化。
- 已实现 `risk_flags` 表。
- 已实现 `RiskFlag` 模型。
- 已实现 `RiskService`。
- 已实现黑名单/提现冻结对提现申请的拦截。
- 已补充 `AuditRiskTest`，覆盖黑名单用户不可提现。
- 已新增轻量防躺平配置：
  - `risk.lie_flat_enabled`
  - `risk.lie_flat_days`

- 已预留置灰恢复规则：充值成功回调确认金额大于等于恢复阈值时恢复返利资格；余额监控只记录活跃，不恢复资格。
