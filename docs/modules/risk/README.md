# Risk 模块：风控与防刷

## 1. 模块目标

负责降低刷量、异常邀请、异常提现和财务风险。

## 2. 模块边界

本模块负责：

- 黑名单。
- 异常邀请检测。
- 最低提现限制。
- 冻结期。
- 风控标记。

本模块不负责：

- 返利计算。
- 支付回调。

## 3. MVP 风控

第一版至少实现：

- 黑名单用户不可提现。
- 最低提现金额。
- 同一事件不重复发放返利。
- 管理员可冻结用户提现。

## 4. 后续风控

- IP / 设备异常。
- 自邀请检测。
- 高频小额充值。
- 批量小号。
- 异常提现。
