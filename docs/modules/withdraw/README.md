# Withdraw 模块：提现申请与审核

## 1. 模块目标

负责用户返利余额提现流程。

## 2. 模块边界

本模块负责：

- 提现申请。
- 余额冻结。
- 管理员审核。
- 打款记录。
- 拒绝后解冻。

本模块不负责：

- 返利计算。
- 支付宝充值。
- 邀请关系。

## 3. MVP 规则

第一版采用人工审核、人工打款：

```text
用户申请提现
-> 系统冻结金额
-> 管理员审核
-> 管理员线下或手动支付宝打款
-> 标记已打款
```

## 4. 状态

```text
pending
approved
paid
rejected
failed
canceled
```

## 5. 核心流程

```text
用户提交提现
-> 校验余额
-> 校验最低提现金额
-> 冻结金额
-> 管理员审核
-> 通过后打款并确认
-> 拒绝后解冻
```
