# Milestone 模块：新人累计充值里程碑奖励

## 1. 模块目标

负责处理新人累计充值达到里程碑后，只奖励直接上级的规则。

## 2. 模块边界

本模块负责：

- 统计被邀请人的累计充值。
- 判断是否达到里程碑。
- 判断奖励次数是否用完。
- 生成里程碑奖励记录。

本模块不负责：

- 正常多级分发。
- 提现。
- 支付宝支付。

## 3. 核心规则

默认规则：

```text
里程碑金额 = 100 元
每次奖励金额 = 15 元
奖励次数 = 2 次
```

示例：

```text
新人累计充值 100 -> 直接上级 +15
新人累计充值 200 -> 直接上级 +15
之后进入正常多级分发
```

跨额规则：

```text
如果 milestone.amount = 100，milestone.max_times = 2
单次充值 250 元最多触发 2 次里程碑
直接上级共获得 30 元
剩余部分不再继续触发里程碑，后续按正常多级返利规则处理
```

## 4. 建议数据结构

| 表 | 说明 |
|---|---|
| user_rebate_progress | 记录被邀请人的里程碑进度 |
| rebate_records | 记录里程碑奖励流水 |

## 5. 核心流程

```text
充值事件
-> 查询被邀请人的直接上级
-> 查询里程碑进度
-> 判断是否触发一个或多个里程碑
-> 写返利流水
-> 更新返利余额
-> 更新里程碑进度
```
