# Sub2API Integration 模块：与 Sub2API 集成

## 1. 模块目标

负责 Sub2Rebate 与现有 Sub2API 的数据和业务衔接。

## 2. 模块边界

本模块负责：

- 读取 Sub2API 用户。
- 识别充值或额度变化。
- 必要时调用 Sub2API 管理接口。
- 适配 Sub2API 表结构。

本模块不负责：

- 返利金额计算。
- 提现。
- 前端页面。

## 3. 推荐方式

第一优先级：

```text
共享 PostgreSQL
读取必要用户和额度数据
通过事件表或监听方式触发返利
```

第二优先级：

```text
Admin API 轮询
```

不推荐：

```text
直接改 Sub2API 核心源码作为第一版方案
```

## 4. 已确认

- 管理员在 Sub2API 手动加额度：需要监控，触发返利。
- Sub2Rebate 自身也有充值/加额度入口，同样触发返利。
- 两个来源都需要被返利引擎识别。
