# Rebate 模块：多级返利与归一化

## 1. 模块目标

负责里程碑奖励结束后的正常多级返利分发。

## 2. 模块边界

本模块负责：

- 获取上级链路。
- 按衰减系数计算各级权重。
- 应用贡献系数。
- 执行归一化。
- 写返利流水。
- 更新返利余额。

本模块不负责：

- 邀请关系绑定。
- 支付回调。
- 提现审核。

## 3. MVP 默认规则

```text
返利池：每充值 100 元，返利池 15 元
分发模式：衰减系数模式
衰减系数：0.4（可配置）
层级限制：无上限，按实际邀请链路所有上级计算（无限级金字塔）
归一化：返利池全部分给有效上级，不留归平台
贡献系数：第一版记录但不启用
```

## 4. 衰减系数算法

```text
第 1 级基础权重 = 1
衰减系数 = decay（默认 0.4）

第 N 级权重 = decay ^ (N - 1)

示例（decay = 0.4）：
第 1 级权重 = 1
第 2 级权重 = 0.4
第 3 级权重 = 0.16
第 4 级权重 = 0.064
第 5 级权重 = 0.0256
...

然后对所有有效上级的权重归一化到 100%
```

优点：

- 层级可动态扩展，不需要手动配置每一级。
- 直接上级永远拿最多，越往上自然递减。
- 只需配置一个衰减系数即可控制分配曲线。

## 5. 核心公式

```text
weight_i = decay ^ (i - 1)
base_amount_i = rebate_pool * weight_i / sum(所有有效上级 weight)
ideal_amount_i = base_amount_i * contribution_factor_i
final_amount_i = ideal_amount_i / sum(所有 ideal_amount) * rebate_pool
```

第一版贡献系数默认 1.0，公式简化为：

```text
final_amount_i = rebate_pool * weight_i / sum(所有有效上级 weight)
```

## 6. 示例

返利池 15 元，衰减系数 0.4，实际上级 3 级：

```text
第 1 级权重 = 1.0
第 2 级权重 = 0.4
第 3 级权重 = 0.16
权重合计 = 1.56

第 1 级 = 15 * 1.0 / 1.56 = 9.62
第 2 级 = 15 * 0.4 / 1.56 = 3.85
第 3 级 = 15 * 0.16 / 1.56 = 1.54
合计 = 15.00（尾差处理归入第 1 级）
```

## 7. 核心流程

```text
充值事件
-> 判断里程碑是否结束
-> 获取完整上级链路（所有上级，无层级上限）
-> 读取衰减系数配置
-> 计算各级权重（weight_i = decay ^ (i-1)）
-> 归一化分配返利池
-> 写 rebate_records
-> 更新 rebate_balances
```
