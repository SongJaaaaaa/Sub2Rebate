# Rebate 模块：多级返利与归一化

## 1. 模块目标

负责里程碑奖励结束后的正常多级返利分发。

## 2. 模块边界

本模块负责：

- 获取上级链路。
- 按衰减系数计算各级权重。
- 应用贡献系数。
- 执行归一化。
- 写返利流水。
- 更新返利余额。

本模块不负责：

- 邀请关系绑定。
- 支付回调。
- 提现审核。

## 3. MVP 默认规则

```text
返利池：每充值 100 元，返利池 15 元
分发模式：衰减系数模式
衰减系数：0.4（可配置）
层级限制：无上限，按实际邀请链路所有上级计算（无限级金字塔）
归一化：返利池全部分给有效上级，不留归平台
贡献系数：第一版记录但不启用
失效节点：不获得新返利；金额处理由配置决定
```

失效节点配置：

```text
rebate.inactive_node_mode = platform
```

可选值：

| 值 | 说明 |
|---|---|
| `platform` | 按原始层级计算，失效节点那一份不发，归平台保留 |
| `exclude_recalculate` | 计算前排除失效节点，对剩余有效上级按衰减系数重新归一化 |

## 4. 衰减系数算法

```text
第 1 级基础权重 = 1
衰减系数 = decay（默认 0.4）

第 N 级权重 = decay ^ (N - 1)

示例（decay = 0.4）：
第 1 级权重 = 1
第 2 级权重 = 0.4
第 3 级权重 = 0.16
第 4 级权重 = 0.064
第 5 级权重 = 0.0256
...

然后对所有有效上级的权重归一化到 100%
```

优点：

- 层级可动态扩展，不需要手动配置每一级。
- 直接上级永远拿最多，越往上自然递减。
- 只需配置一个衰减系数即可控制分配曲线。

## 5. 核心公式

```text
weight_i = decay ^ (i - 1)
base_amount_i = rebate_pool * weight_i / sum(所有有效上级 weight)
ideal_amount_i = base_amount_i * contribution_factor_i
final_amount_i = ideal_amount_i / sum(所有 ideal_amount) * rebate_pool
```

第一版贡献系数默认 1.0，公式简化为：

```text
final_amount_i = rebate_pool * weight_i / sum(所有有效上级 weight)
```

失效节点存在时需要先按 `rebate.inactive_node_mode` 分支处理：

```text
platform:
    使用完整结构链路计算每一级权重。
    rebate_status = disabled 的节点金额不写返利流水，归平台。
    其他有效节点仍保留原始层级。

exclude_recalculate:
    先过滤 rebate_status = disabled 的节点。
    对剩余节点重新编号并重新归一化。
```

## 6. 示例

返利池 15 元，衰减系数 0.4，实际上级 3 级：

```text
第 1 级权重 = 1.0
第 2 级权重 = 0.4
第 3 级权重 = 0.16
权重合计 = 1.56

第 1 级 = 15 * 1.0 / 1.56 = 9.62
第 2 级 = 15 * 0.4 / 1.56 = 3.85
第 3 级 = 15 * 0.16 / 1.56 = 1.54
合计 = 15.00（尾差处理归入第 1 级）
```

## 7. 核心流程

```text
充值事件
-> 判断里程碑是否结束
-> 获取完整上级链路（所有上级，无层级上限）
-> 读取失效节点处理配置
-> 读取衰减系数配置
-> 按配置处理返利失效节点
-> 计算各级权重（weight_i = decay ^ (i-1)）
-> 归一化分配返利池或保留失效节点份额归平台
-> 写 rebate_records
-> 更新 rebate_balances
```

## 8. 已完成

- 模块文档初始化。
- 确定采用衰减系数模式，无限级金字塔结构。
- 已实现 `RebateRecord`、`RebateBalance`、`UserRebateProgress` 基础模型。
- 已实现 `RebateBalanceService::addAvailable()`，供 B6 里程碑奖励更新返利余额。
- `RebateBalanceService::addAvailable()` 已改为按数据库原子自增更新可用余额，避免并发返利入账读改写覆盖。
- 已实现 `DecayRebateCalculator`，按衰减系数计算权重、归一化分配和尾差。
- 已实现 `DecayRebateService`，消费 `rebate_events` 正常返利部分，写返利流水并更新余额。
- 返利事件最终 `processed` 状态由 `ProcessRebateEventJob` 统一标记，里程碑和衰减服务不再各自重复写事件状态。
- 已实现里程碑结束判断：未超过里程碑阈值时不发正常返利。
- 已实现跨额处理：首充 250 元时，里程碑覆盖 200 元，衰减返利只计算剩余 50 元。
- 已补充 `DecayRebateTest`，覆盖三层上级分配、总额守恒、幂等、无上级和 250 元跨额场景。
