# 项目路线图

## 阶段 0：需求与规范

目标：

- 明确项目技术栈。
- 明确目录结构。
- 明确文档规范。
- 明确第一版范围。

状态：进行中。

交付：

- 项目规范文档。
- 架构文档。
- 模块文档。
- 开发流程。
- 项目状态记录。

## 阶段 1：MVP 业务闭环

目标：

完成最小可用的返利闭环。

范围：

- 用户与邀请关系。
- 邀请路径树。
- 新人里程碑奖励。
- 后续多级返利。
- 独立返利余额。
- 人工提现。
- 管理后台配置。
- 完整流水。

不包含：

- 支付宝自动充值。
- 支付宝自动提现。
- 贡献系数自动生效。
- 活动返利。

验收：

- 能模拟用户充值并正确发放返利。
- 能查看返利流水。
- 能申请提现。
- 管理员能审核提现。

## 阶段 2：支付与提现自动化

目标：

让充值和提现逐步自动化。

范围：

- 支付宝扫码充值。
- 支付宝支付回调。
- 支付记录。
- 对账记录。
- 支付宝提现或半自动打款。

验收：

- 支付成功后能自动记录充值。
- 支付回调不会重复处理。
- 提现审核后能进入打款流程。

## 阶段 3：运营增强

目标：

支持大规模推广。

范围：

- 贡献系数。
- 防躺平机制。
- 历史贡献保护。
- 活动返利配置。
- 推广排行榜。
- 数据统计。

验收：

- 活跃推广员能获得更高权重。
- 长期不活跃用户返利降低。
- 返利总额仍受归一化控制。

## 阶段 4：风控与财务完善

目标：

降低刷量和财务风险。

范围：

- 异常邀请检测。
- 小号识别。
- 黑名单。
- 冻结期。
- 导出对账。
- 管理员操作审计。

验收：

- 可识别异常邀请行为。
- 财务可按天/月导出对账。
- 所有敏感操作可追溯。
