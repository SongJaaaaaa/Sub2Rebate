## 阶段 1：MVP 业务闭环

目标：

完成最小可用的返利闭环。

范围：

- 用户与邀请关系。
- 邀请路径树。
- 新人里程碑奖励。
- 后续多级返利。
- 独立返利余额。
- 人工提现。
- 管理后台配置。
- 完整流水。
- 余额调整合并至用户管理页面（弹窗形式），移除独立页面，避免与 API 额度充值功能冲突。
- 推荐关系页面交互增强：支持移动画布、搜索指定用户查看推荐关系、点击节点查看用户信息、大层级性能优化（虚拟渲染/懒加载子树）。
- 推广用户端推荐关系页面：只能查看自己及下级关系，上级不可见。
- 返利配置中心所有配置项增加 Tooltip 提示说明（鼠标悬停显示），说明内容参考根目录方案研讨.md 第 26 节。

不包含：

- 支付宝自动充值（移至阶段 2）。
- 支付宝自动提现。
- 贡献系数自动生效。
- 活动返利。

验收：

- 能模拟用户充值并正确发放返利。
- 能查看返利流水。
- 能申请提现。
- 管理员能审核提现。
- 余额调整在用户管理页面弹窗中完成，不存在独立余额调整页面入口。
- 管理后台推荐关系页面可搜索指定用户、移动画布、点击节点查看详情。
- 推广用户端可查看自己下级推荐关系，无法查看上级信息。
- 返利配置每个字段都有 Tooltip 说明。

## 阶段 2：支付与提现自动化

目标：

让充值和提现逐步自动化。

范围：

- 支付宝扫码充值（推广员/用户端扫码支付增加 API 额度）。
- 支付宝支付回调（回调验签 → 判断支付成功 → 自动增加额度 → 触发返利事件）。
- 支付记录。
- 对账记录。
- 支付宝提现或半自动打款。

验收：

- 推广员可通过扫码支付增加额度。
- 支付成功后能自动记录充值并触发返利。
- 支付回调不会重复处理（幂等）。
- 提现审核后能进入打款流程。

## 阶段 3：运营增强

目标：

支持大规模推广。

范围：

- 贡献系数。
- 防躺平机制。
- 历史贡献保护。
- 活动返利配置。
- 推广排行榜。
- 数据统计。

验收：

- 活跃推广员能获得更高权重。
- 长期不活跃用户返利降低。
- 返利总额仍受归一化控制。

## 阶段 4：风控与财务完善

目标：

降低刷量和财务风险。

范围：

- 异常邀请检测。
- 小号识别。
- 黑名单。
- 冻结期。
- 导出对账。
- 管理员操作审计。

验收：

- 可识别异常邀请行为。
- 财务可按天/月导出对账。
- 所有敏感操作可追溯。
